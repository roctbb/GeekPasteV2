# API для перепроверки задач

Для отправки задачи на перепроверку необходимо выполнить POST-запрос на endpoint `/recheck` с указанием ID кода в query-параметре.

## Endpoint

```
POST /recheck?id={CODE_ID}
```

## Параметры

- **id** (обязательный, query-параметр) — ID кода/решения задачи, которое нужно перепроверить

## Требования

- Пользователь должен быть авторизован
- Пользователь должен иметь роль `teacher` или `admin`
- Код с указанным ID должен существовать
- Код должен быть привязан к задаче (иметь `task_id`)

## Ответ

При успешном выполнении:
- Код: 302 (редирект)
- Перенаправление на `/?id={CODE_ID}`
- Flash-сообщение: "Задача отправлена на повторную проверку."

При ошибке:
- 403 — пользователь не является преподавателем
- 302 с flash-сообщением "Код не найден." — код с указанным ID не существует
- 302 с flash-сообщением "Это не решение задачи." — код не привязан к задаче

## Пример запроса

```bash
curl -X POST 'https://paste.geekclass.ru/recheck?id=ABC123' \
  -H 'Cookie: session=...' \
  -L
```

## Что происходит при перепроверке

1. Сбрасываются результаты предыдущей проверки:
   - `check_state` → `'not checked'`
   - `check_points` → `0`
   - `check_comments` → `None`
   - `checked_at` → `None`

2. Запускается асинхронная задача проверки через Celery

3. Проверка выполняется в зависимости от типа задачи:
   - `tests` — проверка через тесты
   - `gpt` — проверка через GPT
